# Roadmap

Near-term direction after the adoption-focused launch (best-effort sequencing):

1. **`validate-on-add`:** Fail fast when a skill id does not exist in the resolved source (small UX win).
2. **Schema / manifest v2:** Evolve `agentskills.yaml` only with migration notes and semver story for the CLI.
3. **Homebrew tap polish:** Document maintainer-owned tap URLs once `HOMEBREW_TAP_REPO` + formula repo are public.
4. **Editor snippets:** Official snippets or docs for Cursor / VS Code for manifest authoring—not a new IDE plugin yet.
5. **Optional catalog registry:** Signed/indexed skill listings separate from the embedded starter set—defer until demand justifies hosting cost.

Maintainers respond to issues when possible (~best-effort weekly triage); critical fixes prioritized.
